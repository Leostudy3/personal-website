DNSSEC 签名部署方案

DNSSECBINDKSK/ZSKDS Record

完整的 DNSSEC 签名启用流程，从密钥生成到验证链路测试。

背景

DNSSEC（DNS Security Extensions）通过数字签名机制确保 DNS 响应的完整性和真实性，防止 DNS 缓存投毒和中间人攻击。部署 DNSSEC 涉及密钥管理、Zone 签名和 DS 记录注册三个核心环节。

密钥管理架构

KSK（Key Signing Key）

用于签名 ZSK，仅在密钥轮转时使用。建议使用 ECDSA P-256 或 RSA-SHA256 算法，有效期 1 年。KSK 的 DS 记录需要注册到父区。

ZSK（Zone Signing Key）

用于签名 Zone 中的实际 DNS 记录。建议有效期 1 个月，定期轮转。轮转频率决定了签名的新鲜度。

部署步骤

  1. 生成密钥对 — 使用 dnssec-keygen 生成 KSK 和 ZSK
  2. 配置 BIND — 在 named.conf 中启用 auto-dnssec 和 inline-signing
  3. 签名 Zone — 使用 dnssec-signzone 或 BIND 自动签名
  4. 导出 DS 记录 — 从 KSK 生成 DS 记录摘要
  5. 注册 DS 记录 — 在父区提交 DS 记录，等待 TTL 过期
  6. 验证部署 — 使用 dig +dnssec 和在线工具验证签名链路

生成密钥示例

# 生成 KSK
dnssec-keygen -a ECDSAP256SHA256 -b 256 -n ZONE -f KSK example.com

# 生成 ZSK
dnssec-keygen -a ECDSAP256SHA256 -b 256 -n ZONE example.com

# 签名 Zone
dnssec-signzone -A -3 $(head -c 1000 /dev/random | sha1sum | cut -b 1-16) \
  -N INCREMENT -o example.com -t example.com.db

BIND 配置示例

zone "example.com" {
    type master;
    file "dynamic/example.com.db";
    auto-dnssec maintain;
    inline-signing yes;
    key-directory "keys/";
};

验证 DNSSEC

# 检查签名链
dig +dnssec +multi example.com A

# 使用在线工具
# https://dnsviz.net/
# https://dnssec-analyzer.verisignlabs.com/

关键注意事项

TTL 管理：DS 记录的 TTL 直接影响回退速度。修改 DS 记录后需等待原 TTL 过期，建议操作前先将 TTL 降到 300 秒。

时间同步：DNSSEC 签名依赖准确的时间戳，确保 DNS 服务器已配置 NTP 时间同步。

密钥轮转：ZSK 轮转相对简单（预发布→激活→退役），KSK 轮转需提前一个月做预发布，并在父区更新 DS 记录后才可删除旧 KSK。

回退方案：任何 DNSSEC 操作都要准备回退方案——移除 DS 记录 + 重启 named + 清除缓存。
